JAVASCRIPT · ANIMATION

requestAnimationFrame()

Créer des animations fluides et performantes sans bibliothèque externe

Niveau Intermédiaire
Durée ~20 min
Prérequis setInterval, DOM

01 — Pourquoi requestAnimationFrame ?

Avant requestAnimationFrame aussi appelé rAF, les développeurs animaient souvent le DOM avec setInterval ou setTimeout. Ces méthodes fonctionnent, mais elles ne sont pas pensées spécialement pour les animations visuelles.

⚠
Un écran standard se rafraîchit souvent autour de 60 fois par seconde. Si ton setInterval se déclenche toutes les 50ms, l'animation sera peu fluide. S'il se déclenche trop souvent, tu risques de consommer des ressources inutilement.

requestAnimationFrame résout ce problème en disant au navigateur : "appelle ma fonction juste avant le prochain affichage à l'écran". Résultat : l'animation est synchronisée avec le rendu du navigateur.

Critère setInterval requestAnimationFrame
Synchronisation écran ✗ Non ✓ Oui
Adapté aux animations ✗ Moins adapté ✓ Spécialement prévu pour ça
Onglet inactif ✗ Peut être ralenti ou continuer selon les cas ✓ Généralement mis en pause ou fortement optimisé
Consommation CPU/batterie ✗ Moins optimisée pour le rendu visuel ✓ Plus optimisée pour l'animation
Facilité d'arrêt clearInterval(id) cancelAnimationFrame(id)

02 — La syntaxe de base

La fonction requestAnimationFrame() prend en paramètre une callback à exécuter avant le prochain repaint. Elle renvoie un identifiant numérique qu'on peut stocker pour annuler l'animation.

const id = requestAnimationFrame(maFonction);

// Pour annuler :
cancelAnimationFrame(id);
ℹ
Contrairement à setInterval, requestAnimationFrame ne se répète pas tout seul. Pour créer une animation en boucle, la fonction doit se réinscrire elle-même à chaque frame.

Structure minimale

let position = 0;

function animate() {
  // 1. Modifier l'état / déplacer un élément
  position += 5;
  box.style.left = position + "px";

  // 2. Se réinscrire pour la prochaine frame
  requestAnimationFrame(animate);
}

// 3. Premier démarrage
animate();

La fonction animate est appelée une première fois, puis elle se relance elle-même grâce à requestAnimationFrame(animate).

★
Ce premier exemple fonctionne, mais pour une animation plus professionnelle, il vaut mieux utiliser le timestamp fourni automatiquement par requestAnimationFrame. On le voit juste après.

03 — Démarrer et stopper l'animation

Pour stopper une animation rAF, il faut stocker l'ID retourné par requestAnimationFrame() et appeler cancelAnimationFrame(). On peut aussi arrêter naturellement la boucle en ne rappelant pas requestAnimationFrame().

Méthode 1 — Condition d'arrêt dans la boucle

let position = 0;

function animate() {
  if (position >= 200) {
    // On ne rappelle pas requestAnimationFrame
    // Donc la boucle s'arrête naturellement
    return;
  }

  position += 5;
  box.style.left = position + "px";

  requestAnimationFrame(animate);
}

animate();

Méthode 2 — Contrôle externe avec un bouton stop

let rafId = null;
let position = 0;

function animate() {
  position += 2;
  box.style.left = position + "px";

  rafId = requestAnimationFrame(animate);
}

function stop() {
  if (rafId !== null) {
    cancelAnimationFrame(rafId);
    rafId = null;
  }
}

animate();
Démo — Aller simple avec condition d'arrêt
position : 0px

04 — Le paramètre timestamp

La callback passée à requestAnimationFrame() reçoit automatiquement un argument : le timestamp. Il représente un temps en millisecondes, utile pour calculer le temps écoulé entre deux frames.

function animate(timestamp) {
  // timestamp = temps fourni automatiquement par requestAnimationFrame
  console.log(timestamp);

  requestAnimationFrame(animate);
}

// Important : on démarre avec requestAnimationFrame
// pour recevoir un timestamp correct dès le premier appel
requestAnimationFrame(animate);
⚠
Si tu appelles directement animate(), le premier timestamp sera undefined. C'est pour cela qu'il vaut mieux démarrer avec requestAnimationFrame(animate) quand tu utilises le timestamp.

Calculer le delta

Le vrai pattern professionnel consiste à calculer le delta : la durée écoulée depuis la frame précédente. Cela rend l'animation indépendante du nombre de frames par seconde.

let previousTime = null;
let position = 0;
const vitesse = 150; // pixels par seconde

function animate(timestamp) {
  if (previousTime === null) {
    previousTime = timestamp;
  }

  const deltaSeconds = (timestamp - previousTime) / 1000;
  previousTime = timestamp;

  // position = vitesse en px/s × temps écoulé en secondes
  position += vitesse * deltaSeconds;

  box.style.left = position + "px";

  if (position < 300) {
    requestAnimationFrame(animate);
  }
}

requestAnimationFrame(animate);
★
Avec le delta, une animation à 60fps et une animation à 120fps peuvent garder la même vitesse réelle. L'objet parcourt par exemple 150px par seconde, peu importe le nombre de frames.
Démo — Animation basée sur le temps avec delta
Vitesse : 150 px/s — indépendant du fps | 0px

05 — Aller-retour et boucle infinie

Pour inverser la direction, on utilise une variable direction qui bascule entre 1 et -1 quand l'objet touche les bords.

let position  = 0;
let direction = 1; // 1 = droite, -1 = gauche
let rafId     = null;

function animate() {
  position += 4 * direction;

  if (position >= 300) {
    position = 300;
    direction = -1;
  }

  if (position <= 0) {
    position = 0;
    direction = 1;
  }

  box.style.left = position + "px";

  rafId = requestAnimationFrame(animate);
}
Démo — Aller-retour infini
Direction : → | 0px

06 — Comment ça marche en coulisses

  1. Appel initial — Tu démarres l'animation une première fois avec animate() ou avec requestAnimationFrame(animate).
  2. Mise en file — À la fin de la fonction, requestAnimationFrame(animate) demande au navigateur d'exécuter animate avant le prochain affichage.
  3. Repaint — Juste avant d'afficher la prochaine frame, le navigateur exécute les callbacks enregistrées avec requestAnimationFrame.
  4. Boucle — La callback se réinscrit à chaque fois. Si on ne rappelle pas requestAnimationFrame, l'animation s'arrête naturellement.

Résumé visuel du cycle

animate() appelée
  → modifie l'état ou le DOM
  → requestAnimationFrame(animate) → mise en file
                                                    ↓
                              navigateur attend la prochaine frame
                                                    ↓
                                     animate() rappelée ← ─ ─ ┐
                              → modifie l'état ou le DOM          │
                              → requestAnimationFrame(animate) ──┘

07 — Bonnes pratiques

✓ Toujours stocker l'ID

Stocke le retour de requestAnimationFrame dans une variable pour pouvoir annuler l'animation.

let rafId = requestAnimationFrame(animate);

// Plus tard :
cancelAnimationFrame(rafId);

✓ Ne pas lancer plusieurs boucles en même temps

Si l'utilisateur clique plusieurs fois sur "Lancer", plusieurs boucles rAF peuvent tourner en parallèle. Résultat : l'animation peut accélérer ou devenir incontrôlable.

let rafId = null;

function startAnimation() {
  if (rafId !== null) {
    cancelAnimationFrame(rafId);
  }

  rafId = requestAnimationFrame(animate);
}

✓ Utiliser le delta pour la vitesse

Baser la vitesse sur le temps écoulé garantit un comportement plus stable entre différents écrans.

const deltaSeconds = (timestamp - previousTime) / 1000;

position += vitesse * deltaSeconds;

✓ Préférer transform pour les déplacements visuels

Pour déplacer un élément à l'écran, transform est généralement préférable à left, car le navigateur peut souvent mieux optimiser ce type d'animation.

// ❌ Moins performant pour une animation répétée
box.style.left = position + "px";

// ✅ Plus adapté pour une animation fluide
box.style.transform = `translateX(${position}px)`;
⚠
Attention : si ton élément utilise déjà un transform en CSS, par exemple translateY(-50%), alors le JavaScript doit conserver cette transformation. 
// Exemple si l'élément est déjà centré verticalement
box.style.transform = `translate(${position}px, -50%)`;

✓ Éviter de lire et écrire le DOM en boucle

Dans une animation, évite d'alterner sans arrêt lecture du DOM puis écriture du DOM. Cela peut forcer le navigateur à recalculer la mise en page trop souvent.

// Exemple à éviter dans une boucle intensive
const largeur = box.offsetWidth;
box.style.left = largeur + position + "px";

🖥️

😴
OPTIMISÉ

Le navigateur peut mieux gérer les ressources et les onglets inactifs.

⏱️

Le timestamp permet des animations basées sur le temps réel.
