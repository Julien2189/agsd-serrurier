Installation & environnement

DÉPART

Installer Python, utiliser le terminal, créer un environnement virtuel et organiser un projet.

Avant de coder, vérifie que Python est installé et accessible depuis le terminal.

# Terminal
python --version
py --version
pip --version

# Lancer un fichier
python main.py

# Lancer une commande rapide
python -c "print('Bonjour Python')"

Un environnement virtuel évite les conflits entre projets.

# Créer le venv
python -m venv .venv

# Activer sur Windows
.venv\Scripts\activate

# Activer sur Linux / macOS
source .venv/bin/activate

# Installer une dépendance
pip install requests

# Sauvegarder les dépendances
pip freeze > requirements.txt

# Réinstaller les dépendances
pip install -r requirements.txt

Pour progresser, il faut apprendre à organiser son code en fichiers simples.

Variables & types

BASES

Les bases indispensables : variables, nombres, booléens, None, print et input.

En Python, il n'y a pas de let, const ou var. On affecte directement une valeur à un nom.

nom = "Julien"
age = 40
prix = 19.99
actif = True

x, y, z = 1, 2, 3
a = b = c = 0

MAX_RETRY = 5  # constante par convention

Python détecte le type automatiquement, mais tu peux le vérifier avec type() et isinstance().

age = 40
prix = 29.99
nom = "Julien"
connecte = True
resultat = None

print(type(prix))       # <class 'float'>
print(type(nom))        # <class 'str'>
print(type(connecte))   # <class 'bool'>

print(isinstance(age, int))

None représente l'absence de valeur. On compare avec is ou is not.

if resultat is None:
    print("Aucun résultat")

if resultat is not None:
    print("Résultat trouvé")

Fonctions

FONCTIONS

Créer du code réutilisable.

Une fonction permet de regrouper une logique et de la réutiliser.

def saluer(nom):
    return f"Bonjour {nom}"

message = saluer("Julien")
print(message)

def addition(a, b):
    return a + b

print(addition(2, 3))

*args récupère plusieurs arguments positionnels. **kwargs récupère plusieurs arguments nommés.

def somme(*args):
    return sum(args)

print(somme(1, 2, 3, 4))

def afficher_config(**kwargs):
    for cle, valeur in kwargs.items():
        print(f"{cle} = {valeur}")

afficher_config(host="localhost", port=3000)

Python cherche les variables selon la règle LEGB : Local, Enclosing, Global, Built-in.

compteur = 0

def incrementer():
    global compteur
    compteur += 1

incrementer()
print(compteur)

def creer_compteur():
    count = 0

    def ajouter():
        nonlocal count
        count += 1
        return count

    return ajouter

c = creer_compteur()
print(c())
print(c())

Une lambda est utile pour une petite fonction courte, souvent dans sorted, map ou filter.

carre = lambda x: x ** 2
print(carre(5))

users = [
    {"nom": "Bob", "age": 30},
    {"nom": "Alice", "age": 25}
]

tries = sorted(users, key=lambda user: user["age"])
print(tries)

Une closure garde accès aux variables de la fonction externe.

def multiplier_par(n):
    def multiplier(x):
        return x * n

    return multiplier

double = multiplier_par(2)
triple = multiplier_par(3)

print(double(10))
print(triple(10))

Décorateurs

FONCTIONS

Modifier le comportement d'une fonction avec @.

Un décorateur reçoit une fonction et retourne une fonction modifiée.

from functools import wraps
import time

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        duree = time.time() - start
        print(f"{func.__name__} : {duree:.3f}s")
        return result

    return wrapper

@timer
def traitement():
    time.sleep(1)

traitement()

@property permet d'utiliser une méthode comme un attribut.

class Cercle:
    def __init__(self, rayon):
        self._rayon = rayon

    @property
    def rayon(self):
        return self._rayon

    @rayon.setter
    def rayon(self, valeur):
        if valeur < 0:
            raise ValueError("Rayon négatif")
        self._rayon = valeur

    @property
    def aire(self):
        return 3.14159 * self._rayon ** 2

c = Cercle(5)
print(c.aire)
c.rayon = 10
print(c.aire)

POO : classes & objets

POO

Créer ses propres types avec des classes.

self représente l'objet courant, comme this en JavaScript.

class User:
    def __init__(self, nom, email):
        self.nom = nom
        self.email = email
        self.actif = True

    def saluer(self):
        return f"Salut, je suis {self.nom}"

    def __repr__(self):
        return f"User(nom={self.nom!r})"

u = User("Julien", "julien@example.com")
print(u.saluer())
print(u)

Les méthodes dunder permettent de personnaliser les opérateurs et fonctions intégrées.

class Vecteur:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Vecteur({self.x}, {self.y})"

    def __add__(self, other):
        return Vecteur(self.x + other.x, self.y + other.y)

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

v1 = Vecteur(1, 2)
v2 = Vecteur(3, 4)

print(v1 + v2)
print(v1 == v2)

Héritage

POO

Réutiliser et spécialiser des classes.

Une classe enfant hérite des méthodes et attributs d'une classe parent.

class Animal:
    def __init__(self, nom):
        self.nom = nom

    def parler(self):
        raise NotImplementedError

class Chien(Animal):
    def __init__(self, nom, race):
        super().__init__(nom)
        self.race = race

    def parler(self):
        return f"{self.nom} dit Woof !"

rex = Chien("Rex", "Berger")
print(rex.parler())
print(isinstance(rex, Animal))

Une classe abstraite définit un contrat que les classes enfants doivent respecter.

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def aire(self) -> float:
        pass

class Carre(Shape):
    def __init__(self, cote):
        self.cote = cote

    def aire(self):
        return self.cote ** 2

c = Carre(5)
print(c.aire())

Un mixin ajoute une fonctionnalité sans représenter forcément une relation parent/enfant forte.

import json

class JsonMixin:

class User(JsonMixin):
    def __init__(self, nom):
        self.nom = nom

u = User("Julien")
print(u.to_json())

Générateurs & itérateurs

AVANCÉ

Produire des valeurs à la demande.

yield produit une valeur puis met la fonction en pause.

def compter(maximum):
    n = 0
    while n < maximum:
        yield n
        n += 1

for valeur in compter(5):
    print(valeur)

yield from permet de déléguer à un autre générateur.

def aplatir(elements):
    for item in elements:
        if isinstance(item, list):
            yield from aplatir(item)
        else:
            yield item

resultat = list(aplatir([1, [2, [3, 4]], 5]))
print(resultat)

from itertools import chain, combinations, product, islice, count

print(list(product([0, 1], repeat=3)))
print(list(islice(count(), 5)))

Async / await

AVANCÉ

Concurrence pour les opérations I/O.

asyncio est utile pour les tâches réseau, fichiers ou attentes, pas pour le CPU intensif.

import asyncio

async def fetch_data(url: str) -> str:
    print(f"Début : {url}")
    await asyncio.sleep(1)
    return f"Données de {url}"

async def main():
    result = await fetch_data("https://api.example.com")
    print(result)

asyncio.run(main())

gather lance plusieurs coroutines et attend tous les résultats.

import asyncio

async def wait_and_return(n):
    await asyncio.sleep(1)
    return n

async def main():
    resultats = await asyncio.gather(
        wait_and_return(1),
        wait_and_return(2),
        wait_and_return(3),
    )
    print(resultats)

asyncio.run(main())

TaskGroup est une approche moderne pour gérer plusieurs tâches async.

import asyncio

async def job(n):
    await asyncio.sleep(1)
    return n * 2

async def main():
    async with asyncio.TaskGroup() as tg:
        t1 = tg.create_task(job(10))
        t2 = tg.create_task(job(20))

    print(t1.result())
    print(t2.result())

asyncio.run(main())

Regex

INTERMÉDIAIRE

Rechercher et transformer du texte avec le module re.

Utilise toujours des raw strings r'...' pour écrire des regex.

import re

texte = "Prix : 42 euros"

    print(m.group())

print(re.match(r"\d+", "42 ans"))
print(re.fullmatch(r"\d{5}", "75001"))

Les regex sont puissantes mais peuvent devenir difficiles à lire. Préfère des patterns simples.

# \d       chiffre
# \w       lettre, chiffre ou underscore
# \s       espace
# .        n'importe quel caractère sauf retour ligne
# +        une ou plusieurs fois
# *        zéro ou plusieurs fois
# ?        zéro ou une fois
# {n}      exactement n fois
# ^        début
# $        fin

email = r"^[\w.+-]+@[\w-]+\.[\w.-]+$"
tel_fr = r"^0[67]\d{8}$"

Les groupes nommés rendent les regex plus lisibles.

import re

texte = "Date : 22/03/2026"
pattern = r"(?P<jour>\d{2})/(?P<mois>\d{2})/(?P<annee>\d{4})"

m = re.search(pattern, texte)

if m:
    print(m.group("jour"))
    print(m.group("mois"))
    print(m.group("annee"))
    print(m.groupdict())

Functools & contextlib

INTERMÉDIAIRE

Outils avancés pour fonctions et context managers.

lru_cache mémorise les résultats pour accélérer les appels répétés.

from functools import lru_cache

@lru_cache(maxsize=128)
def fib(n):
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(100))
print(fib.cache_info())

partial crée une nouvelle fonction avec certains arguments déjà fixés.

from functools import partial

def puissance(base, exp):
    return base ** exp

carre = partial(puissance, exp=2)
cube = partial(puissance, exp=3)

print(carre(5))
print(cube(3))

Le code avant yield prépare, le code après yield nettoie.

from contextlib import contextmanager
import time

@contextmanager
def timer(label):
    start = time.perf_counter()
    try:
        yield
    finally:
        elapsed = time.perf_counter() - start
        print(f"{label} : {elapsed:.3f}s")

with timer("Calcul"):
    sum(range(1_000_000))

Threading & multiprocessing

OUTILS

Concurrence et parallélisme.

Dans CPython classique, les threads sont surtout utiles pour l'I/O. Pour le CPU, on préfère souvent les processus.

from concurrent.futures import ThreadPoolExecutor
import time

def fetch(url):
    time.sleep(1)
    return f"Données de {url}"

urls = ["url1", "url2", "url3"]

with ThreadPoolExecutor(max_workers=3) as pool:
    resultats = list(pool.map(fetch, urls))

print(resultats)

Le multiprocessing lance de vrais processus séparés.

from concurrent.futures import ProcessPoolExecutor

def calcul_lourd(n):
    return sum(i * i for i in range(n))

if __name__ == "__main__":
    with ProcessPoolExecutor() as pool:
        resultats = list(pool.map(calcul_lourd, [1_000_000] * 4))

    print(resultats)

Lock protège une ressource partagée. Queue permet la communication thread-safe.

import threading
from queue import Queue

lock = threading.Lock()
compteur = 0

def incrementer():
    global compteur
    with lock:
        compteur += 1

q = Queue()
q.put("tâche 1")
q.put("tâche 2")

while not q.empty():
    print(q.get())
    q.task_done()

Les fixtures préparent des données. parametrize teste plusieurs cas.

    assert a + b == expected

Un mock simule une dépendance externe comme une API.

    return response.json()["name"]

    mock_get.assert_called_once()
